# 系统架构

## 总体架构

```text
Boss / 招聘网站页面
        |
        v
Chrome Extension
  - DOM 采集
  - 侧边栏分析
  - 一键保存
  - 辅助填充
        |
        v
Local API Service
  - 岗位解析
  - AI 编排
  - 简历生成
  - 投递记录
        |
        v
SQLite Local Database
  - 经历库
  - 岗位池
  - 公司记录
  - 简历版本
  - 沟通记录

Web App
  - 经历库管理
  - 岗位看板
  - 投递包预览
  - 统计复盘
```

## 模块边界

### Browser Extension

职责：

- 读取当前招聘页面可见信息。
- 将岗位和公司字段标准化后发送给本地 API。
- 展示当前岗位的匹配评分、风险提示和推荐动作。
- 将生成的打招呼语填入页面输入框，由用户确认发送。

不负责：

- 存储长期数据。
- 直接生成简历文件。
- 绕过平台限制或隐藏自动化行为。

### Web App

职责：

- 管理经历库、技能证据、简历模板。
- 管理岗位池、公司记录和投递状态。
- 查看 AI 分析结果和投递包。
- 维护用户偏好、黑名单、自动化规则。
- 展示统计和复盘。

### Local API

职责：

- 提供本地 HTTP API。
- 调用数据库和 AI Provider。
- 执行匹配评分、风险识别、文档生成。
- 管理后台任务队列。

### AI Layer

职责：

- JD 解析。
- 公司偏好分析。
- 经历素材检索和排序。
- 个性化简历生成。
- 打招呼语、跟进语和面试准备生成。
- 输出结构化 JSON 并通过 Zod 校验。

### Database

职责：

- 保存用户经历库。
- 保存岗位、公司、联系人和投递记录。
- 保存每次 AI 生成的简历版本和话术。
- 支持全文检索和语义检索。

## 核心数据流

### 岗位采集

```text
用户打开 Boss 岗位页面
 -> 插件提取页面字段
 -> 本地 API 标准化
 -> 数据库存储 job/company/recruiter
 -> AI 分析 JD 和风险
 -> Web/插件展示评分
```

### 定制简历生成

```text
选择岗位
 -> AI 解析 JD 必须项和加分项
 -> 检索经历库
 -> 选择最相关经历素材
 -> 生成简历大纲
 -> 生成 Markdown/DOCX/PDF
 -> 保存 resume_version
 -> 用户预览确认
```

### 投递跟进

```text
生成打招呼语
 -> 插件辅助填入
 -> 用户确认发送
 -> 记录 application_event
 -> 到期提醒跟进
 -> 记录回复和结果
 -> 统计版本效果
```

## 自动化边界

- 默认由用户确认发送消息和投递简历。
- 允许用户配置低风险岗位的自动化规则，但必须可审计、可撤销。
- 所有自动生成内容需要保留来源岗位、使用经历、生成时间和模型信息。
