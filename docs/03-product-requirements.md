# 产品需求

## 用户画像

目标用户是希望提高求职投递效率的人，尤其适合：

- 有多个项目、实习或工作经历，需要针对岗位重组简历的人。
- 在 Boss 直聘等平台高频筛选岗位的人。
- 希望 AI 帮助判断岗位质量和生成沟通材料的人。
- 不想手工维护大量投递记录的人。

## 核心对象

- 经历：项目、实习、工作、开源、校园经历、作品。
- 技能证据：某项技能对应的真实经历和可讲细节。
- 岗位：招聘页面采集的职位信息。
- 公司：公司背景、风险信号和偏好。
- 投递包：简历、打招呼语、跟进语、面试准备。
- 投递记录：状态、时间线、回复、结果。

## MVP 功能

### P0

- 经历库 CRUD。
- 用户求职目标和偏好配置。
- Boss 当前页面岗位采集。
- JD 结构化解析。
- 岗位匹配评分。
- 风险识别。
- 个性化打招呼语生成。
- 个性化简历 Markdown 生成。
- 投递状态看板。

### P1

- DOCX/PDF 简历导出。
- 插件侧边栏实时展示分析。
- 一键填入打招呼语。
- 跟进提醒。
- 面试准备包。
- 简历版本效果统计。

### P2

- 半自动批量浏览岗位。
- 自动跳过黑名单岗位。
- 多平台适配。
- 语义向量检索。
- 自动 A/B 测试打招呼语。
- 本地模型支持。

## 岗位评分

默认评分维度：

| 维度         | 权重 |
| ------------ | ---- |
| 技能匹配     | 30   |
| 项目经验匹配 | 20   |
| 薪资匹配     | 15   |
| 行业匹配     | 10   |
| 城市/通勤    | 10   |
| 公司质量     | 10   |
| 风险扣分     | -20  |

输出：

- 总分。
- 推荐动作：优先投递、可投递、谨慎沟通、跳过。
- 匹配原因。
- 风险点。
- 简历强化建议。

## 投递包内容

每个岗位可以生成：

- Boss 打招呼语。
- 1 页快投版简历。
- 2 页正式版简历。
- 技术负责人版简历。
- HR 简短介绍。
- 面试自我介绍。
- 岗位匹配说明。
- 面试问题预测。
- 反问问题清单。
