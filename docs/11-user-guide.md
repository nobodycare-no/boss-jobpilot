# boss-jobpilot 用户使用手册

本文档说明当前公开版本的主要使用方式。项目仍然以本地个人使用为主，不会自动投递岗位，也不会自动向招聘方发送消息。

## 当前能力

- 启动本地 API 和 Web 工作台。
- 维护简历素材，包括结构化经历和整份简历文本。
- 手动保存岗位，或通过 Chrome 插件从 Boss 直聘岗位页采集当前岗位。
- 对岗位进行匹配分析，查看匹配分数、关键技能、风险信号和投递建议。
- 基于岗位和简历素材生成 Markdown 定制简历草稿。
- 基于岗位、分析结果和简历素材生成打招呼语。
- 在岗位池中按核心阶段筛选岗位。
- 记录打招呼、投递、回复、面试等状态，并设置下一次跟进时间。
- 复制单项草稿或完整投递包，方便手动留档和投递。
- 查看 AI Provider 状态和最近生成记录。

当前版本不支持：

- 自动批量浏览岗位列表。
- 自动点击投递或自动发送消息。
- 完整在线简历编辑器、DOCX/PDF 导出。
- 云端同步或多人协作。

## 启动项目

Windows 用户可以在项目根目录双击：

```text
start.bat
```

脚本会启动：

- API：`http://127.0.0.1:4000`
- Web：`http://127.0.0.1:5173`

开发者也可以在项目根目录运行：

```powershell
npm run dev
```

单独启动：

```powershell
npm run dev:api
npm run dev:web
```

插件开发：

```powershell
npm run dev:extension
```

完整检查：

```powershell
npm run typecheck
npm run test
npm run lint
npm run build
```

## 配置 AI Provider

项目支持 OpenAI-compatible API。复制 `.env.example` 为 `.env` 后填写：

```text
AI_API_KEY=your_api_key
AI_API_BASE_URL=https://your-provider.example/v1
AI_MODEL=your-model-id
```

修改 `.env` 后需要重启 API。只刷新浏览器页面不会让后端重新读取密钥。

不配置 `AI_API_KEY` 时，系统会使用本地规则版逻辑。配置真实 Provider 后，岗位分析、定制简历、打招呼语和策略复盘会优先调用模型；调用失败时，本次操作会回退到规则版结果，并在界面展示降级原因。

## 数据位置

默认数据库：

```text
data/jobpilot.sqlite
```

可通过环境变量覆盖：

```powershell
$env:DATABASE_PATH="D:\path\jobpilot.sqlite"
npm run dev:api
```

数据库、临时文件和 `.env` 不应提交到 Git。

## 使用 Web 工作台

打开：

```text
http://127.0.0.1:5173
```

### 维护简历素材

简历素材是岗位分析和内容生成的基础。建议先补充：

- 整份简历文本：用于快速提供完整背景。
- 项目、实习、工作、开源或教育经历：用于更精确地匹配岗位。
- 技术栈、职责、成果、量化指标和证据等级：用于提高生成内容的可信度。

标记为不使用的素材不会参与岗位匹配和简历生成。

### 保存岗位

可以在 Web 工作台手动填写岗位信息，也可以用 Chrome 插件从 Boss 直聘岗位页保存当前岗位。

岗位通常包含：

- 平台、城市、岗位名称、公司。
- 薪资、经验要求、学历要求。
- 岗位链接和 JD 原文。

保存后，岗位会进入岗位池。删除岗位会同时清理该岗位关联的分析记录、简历版本、打招呼语和状态事件。

### 使用岗位池

岗位池只保留核心筛选入口，重点帮助你判断下一步应该处理什么：

- 全部：查看已保存的全部岗位。
- 待生成简历：还没有定制简历的岗位。
- 简历已生成：已经有定制简历，但还没有打招呼语的岗位。
- 话术待发送：已经生成打招呼语，等待手动复制和发送。
- 已打招呼：已经联系招聘方。
- 已投递后：已经进入投递、回复、面试、Offer、拒绝或关闭阶段。
- 待跟进：下一次跟进时间已经到期或逾期的岗位。

如果岗位较多，可以通过分页查看完整岗位池。

### 分析岗位

在岗位卡片上点击分析入口后，系统会生成并保存：

- 匹配分数。
- 投递建议。
- 匹配关键词。
- 必需技能和加分技能。
- 匹配到的简历素材。
- 风险信号。
- 简历策略。

如果还没有可用简历素材，建议先补充素材再分析。

### 生成定制简历

已分析岗位可以生成定制简历草稿。系统会基于：

- 当前岗位信息。
- 最新岗位分析结果。
- 匹配到的简历素材。
- 已添加的整份简历文本和结构化经历。

生成结果会保存为该岗位的简历版本。手动编辑后保存会创建新版本，不会覆盖原版本。

### 生成打招呼语

已分析岗位可以生成打招呼语草稿。生成结果只保存在本地，不会自动发送给 Boss 直聘页面。

复制或填入页面后，请先检查内容，再由你手动点击发送。

### 复制投递包

岗位卡片可以复制完整投递包，通常包括：

- 岗位信息和 JD。
- 岗位分析。
- 匹配经历。
- 定制简历 Markdown。
- 打招呼语。
- 投递状态和状态时间线。

缺失内容会以明确占位说明保留，方便后续补齐。

### 标记投递状态

生成打招呼语后，可以在岗位卡片中标记：

- 已打招呼。
- 已投递。
- 已回复。
- 面试。
- Offer。
- 已拒绝。
- 关闭。

这些状态只更新本地记录，不会操作招聘网站。

### 设置跟进时间

设置下一次跟进时间后，到期或逾期岗位会出现在“待跟进”分组中。若不再需要跟进，可以清除跟进时间。

## 使用 Chrome 插件

先确保本地 API 已启动：

```text
http://127.0.0.1:4000
```

构建或启动插件：

```powershell
npm run dev:extension
```

或：

```powershell
corepack pnpm --filter @boss-jobpilot/extension build
```

在 Chrome 打开：

```text
chrome://extensions
```

启用开发者模式，加载：

```text
apps/extension/build/chrome-mv3-dev
```

或：

```text
apps/extension/build/chrome-mv3-prod
```

在 Boss 直聘岗位页点击插件图标，可以：

- 保存当前岗位到本地岗位池。
- 读取本地打招呼语并复制。
- 尝试把打招呼语填入当前页面输入框。
- 读取并复制完整投递包。

插件只会填入内容，不会自动点击发送。

## 推荐工作流

1. 启动 `start.bat` 或 `npm run dev`。
2. 在简历素材区添加整份简历和关键经历。
3. 打开 Boss 直聘岗位页，用插件保存岗位。
4. 回到 Web 工作台，在岗位池中分析岗位。
5. 查看匹配分数、风险和匹配素材。
6. 生成定制简历。
7. 生成打招呼语。
8. 复制完整投递包留档。
9. 回到 Boss 页面，用插件读取并复制或填入打招呼语。
10. 检查内容后手动发送或投递。
11. 回到岗位卡片标记状态。
12. 设置下一次跟进时间。
13. 使用岗位池分组和分页持续推进。

## 常见问题

### 插件提示本地 API 不可用

先确认：

```text
http://127.0.0.1:4000/health
```

如果无法访问，重新运行：

```powershell
npm run dev
```

### AI Provider 显示未配置

检查：

- `.env` 是否在项目根目录。
- 是否填写了 `AI_API_KEY`。
- API 是否已经重启。
- Web 是否连接到同一个 API 地址。

### AI Provider 验证失败

优先检查：

- `AI_MODEL` 是否是 Provider 可用的完整模型 ID。
- `AI_API_BASE_URL` 是否是兼容接口根路径。
- 本机网络或代理是否可以访问 Provider。
- 修改 `.env` 后是否已重启 API。

### 生成简历或打招呼语前提示需要分析岗位

先在岗位卡片中完成岗位分析，再生成简历或打招呼语。

### 生成结果不理想

优先补充简历素材，尤其是技术栈、职责、成果、指标和证据等级。岗位 JD 本身信息不足时，生成内容也会更保守。

### 保存岗位后没有出现在岗位池

先确认当前岗位池筛选是否为“全部”。如果仍未出现，刷新页面并检查本地 API 是否仍在运行。
