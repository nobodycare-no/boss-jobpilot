# 当前软件使用手册

本文档记录 `boss-jobpilot` 当前版本的实际可用功能。后续每次迭代新增或改变用户路径时，都应同步更新本文档。

## 当前能力边界

当前版本已经支持：

- 本地启动 API 和 Web 工作台。
- 维护个人经历库。
- 手动保存岗位。
- 使用 Chrome 插件在 Boss 直聘岗位页一键采集当前岗位。
- 对岗位进行规则版匹配分析。
- 将岗位分析结果保存为历史记录。
- 基于最新岗位分析和匹配经历生成 Markdown 定制简历草稿。
- 基于最新岗位分析和匹配经历生成个性化打招呼语草稿。
- 一键复制 Markdown 简历草稿和打招呼语草稿。
- 一键复制完整投递包 Markdown 文本。
- 在岗位池中按投递状态查看数量并筛选岗位。
- 在岗位卡片中手动标记投递状态：已打招呼、已投递、已回复、面试、Offer、已拒绝、关闭。
- 在岗位卡片中查看投递状态变更时间线。

当前版本尚未支持：

- 自动批量浏览 Boss 岗位列表。
- 自动打招呼或自动投递。
- 真实 AI 模型生成，当前分析和简历生成仍是规则版。
- 简历在线编辑器和导出 PDF/DOCX。
- 自动发送打招呼语或自动投递。
- 跟进提醒、完整投递复盘和面试准备闭环。

## 一键启动

Windows 新手用户可以在项目根目录双击：

```text
start.bat
```

脚本会检查 Node/Corepack，必要时安装依赖，并启动：

- API：`http://127.0.0.1:4000`
- Web：`http://127.0.0.1:5173`

关闭项目时，关闭 `boss-jobpilot API` 和 `boss-jobpilot Web` 两个命令行窗口即可。

## 程序员启动

在项目根目录运行：

```powershell
npm run dev
```

该命令只启动 API 和 Web，不启动插件。

单独启动：

```powershell
npm run dev:api
npm run dev:web
```

插件开发启动：

```powershell
npm run dev:extension
```

完整验证：

```powershell
npm run typecheck
npm run test
npm run lint
npm run build
```

## 数据保存位置

默认数据文件：

```text
data/jobpilot.sqlite
```

可以通过环境变量覆盖：

```powershell
$env:DATABASE_PATH="D:\path\jobpilot.sqlite"
npm run dev:api
```

## 使用 Web 工作台

打开：

```text
http://127.0.0.1:5173
```

### 维护经历库

在经历库中录入你的项目、实习、工作、开源或教育经历。建议尽量填写：

- 标题：项目或经历名称。
- 组织：公司、学校、团队或项目来源。
- 角色：你在经历中的角色。
- 技术栈：用于岗位匹配。
- 职责：你具体做了什么。
- 成果：交付结果或业务影响。
- 指标：量化结果。
- 证据等级：是否能深聊，或是否不能使用。
- 负责程度：主导、负责、参与或协助。

`do_not_use` 的经历不会参与岗位匹配和简历生成。

### 保存岗位

可以在 Web 的岗位采集表单中手动填写：

- 平台。
- 城市。
- 岗位标题。
- 公司。
- 薪资。
- 经验要求。
- 学历要求。
- 链接。
- JD 原文。

点击“保存岗位”后，岗位会进入岗位池。

在岗位卡片右上角点击删除图标，可以删除已保存岗位。删除岗位会同时清理该岗位关联的分析记录、简历版本、打招呼语草稿和投递状态事件。

### 使用投递看板

岗位列表上方会显示投递状态看板：

- 全部。
- 未生成草稿。
- 草稿。
- 已打招呼。
- 已投递。
- 已回复。
- 面试中。
- Offer。
- 已拒绝。
- 已关闭。

点击任一状态会筛选岗位列表，只显示该状态下的岗位。再次点击“全部”可回到完整岗位池。

### 分析岗位

在岗位卡片右上角点击图表图标。

系统会生成并保存：

- 匹配分数。
- 投递建议。
- 匹配关键词。
- 必需技能。
- 加分技能。
- 匹配经历标题、摘要和技术栈。
- 风险信号。
- 简历策略。

### 生成定制简历

在已经分析过的岗位卡片右上角点击文档图标。

系统会基于：

- 当前岗位信息。
- 最新岗位分析。
- 匹配到的经历。

生成 Markdown 简历草稿，并保存为该岗位的简历版本。岗位卡片会显示：

- 定制说明。
- 变更摘要。
- Markdown 内容预览。

点击简历草稿面板右上角的“复制”按钮，可以把完整 Markdown 简历复制到剪贴板。

如果还没有分析岗位，系统会提示先进行岗位分析。

### 生成打招呼语

在已经分析过的岗位卡片右上角点击消息图标。

系统会基于：

- 当前岗位信息。
- 最新岗位分析。
- 匹配到的经历。
- 最新定制简历版本，如果已经生成。

生成一条短打招呼语，并保存为该岗位的 `application draft`。岗位卡片会显示：

- 打招呼语草稿。
- 草稿状态。
- 已关联的简历版本 ID，如果生成时已有简历版本。

点击打招呼语草稿面板右上角的“复制”按钮，可以把话术复制到剪贴板。

当前版本只生成草稿，不会自动发送给招聘者。

### 复制完整投递包

在岗位卡片右上角点击复制图标，可以把当前岗位的完整投递包复制到剪贴板。

投递包会按 Markdown 组织当前已有内容：

- 岗位信息和 JD。
- 岗位分析、匹配分、技能和风险信号。
- 匹配经历详情。
- 定制简历元信息和 Markdown 简历正文。
- 打招呼语、投递状态和状态时间线。

如果某部分尚未生成，投递包会保留明确占位说明，便于后续补齐。

### 标记投递状态

生成打招呼语后，岗位卡片的打招呼语草稿面板会出现状态按钮：

- 已打招呼。
- 已投递。
- 已回复。
- 面试。
- Offer。
- 已拒绝。
- 关闭。

点击按钮只会更新本地投递记录，不会操作 Boss 页面，也不会自动发送消息。

当状态从一个阶段切换到另一个阶段时，系统会在本地 `application_events` 中记录一次状态变更事件。标记为“已投递”时，如果还没有投递时间，系统会自动写入当前时间。

状态变更后，打招呼语草稿面板底部会出现“状态时间线”。时间线会按发生时间展示每次状态变化，例如从“草稿”更新为“已打招呼”、从“已投递”更新为“面试中”。

## 使用 Chrome 插件采集 Boss 岗位

先确保本地 API 已启动：

```text
http://127.0.0.1:4000
```

构建或启动插件：

```powershell
npm run dev:extension
```

或生产构建：

```powershell
corepack pnpm --filter @boss-jobpilot/extension build
```

打开 Chrome：

```text
chrome://extensions
```

开启“开发者模式”，点击“加载已解压的扩展程序”。

开发模式通常加载：

```text
D:\projects\boss\apps\extension\build\chrome-mv3-dev
```

生产构建加载：

```text
D:\projects\boss\apps\extension\build\chrome-mv3-prod
```

打开 Boss 直聘岗位详情页：

```text
https://www.zhipin.com/...
```

点击 Chrome 右上角 `Boss Jobpilot` 插件图标，再点击“保存当前岗位”。保存成功后，回到 Web 工作台刷新岗位池即可看到岗位。

## 推荐工作流

1. 启动 `start.bat` 或 `npm run dev`。
2. 在经历库录入所有可用经历。
3. 打开 Boss 岗位详情页，用插件保存岗位。
4. 回到 Web 岗位池，点击“分析”。
5. 查看匹配分数、风险和匹配经历。
6. 点击“生成定制简历”。
7. 点击“生成打招呼语”。
8. 点击岗位卡片右上角的复制图标，复制完整投递包留档；需要单独投递时，也可点击草稿面板的“复制”按钮复制打招呼语或 Markdown 简历。
9. 手动完成打招呼或投递后，在岗位卡片中标记对应状态。
10. 使用投递看板按状态检查待推进岗位。

## 故障排查

### 插件点击后提示保存失败

通常是本地 API 没启动。先确认：

```text
http://127.0.0.1:4000/health
```

如果无法访问，重新运行：

```powershell
npm run dev
```

### Chrome 加载插件后没有按钮

确认加载的是构建产物目录，不是源码目录：

```text
apps/extension/build/chrome-mv3-prod
```

或开发模式目录：

```text
apps/extension/build/chrome-mv3-dev
```

### 生成简历提示需要先分析岗位

先点击岗位卡片右上角的图表图标进行分析，再点击文档图标生成简历。

### 生成打招呼语提示需要先分析岗位

先点击岗位卡片右上角的图表图标进行分析，再点击消息图标生成打招呼语。

### 匹配经历仍显示为 ID

如果岗位分析结果里的某个经历 ID 在当前经历库中找不到，系统会退回显示原始 ID。通常是该经历后来被删除或当前数据文件不是生成分析时的数据文件。

### 删除岗位提示失败

当前版本会在删除岗位时同步清理该岗位下的分析、简历、投递草稿和事件。如果仍然失败，先刷新岗位池确认该岗位是否已经被其他窗口删除，再检查本地 API 是否仍在运行。
