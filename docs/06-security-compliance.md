# 安全与合规

## 数据安全

本项目处理高敏感个人数据，包括：

- 简历
- 联系方式
- 求职偏好
- 投递记录
- 公司沟通内容
- 薪资预期

默认策略：

- 数据保存在本地 SQLite。
- 不默认上传到自有服务器。
- 调用云模型前明确展示将发送的字段类型。
- 支持删除所有本地数据。
- 支持导出和备份。

## 平台合规

招聘平台可能限制自动化行为。项目需要避免：

- 高频批量访问。
- 绕过登录、验证码或风控。
- 模拟异常用户行为。
- 未经用户确认自动发送大量消息。

建议策略：

- MVP 只做当前页面采集和辅助填写。
- 自动化等级默认 L2/L3。
- L4/L5 需要用户显式开启，并配置频率、条件和上限。
- 所有自动化动作留日志。

## AI 合规

生成内容必须：

- 基于用户真实经历。
- 保留改写说明。
- 标记低置信度内容。
- 不生成虚假学历、工作履历或项目成果。

## 风险控制

高风险动作包括：

- 自动发送打招呼语。
- 自动投递简历。
- 自动跟进。
- 将完整简历发送给模型服务。

这些动作需要：

- 用户授权。
- 频率限制。
- 可回滚记录。
- 失败重试上限。
