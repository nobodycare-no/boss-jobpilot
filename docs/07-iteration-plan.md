# 迭代计划

## 里程碑 0：项目脚手架

目标：

- 初始化 Git 仓库。
- 创建 pnpm workspace。
- 创建 apps/web、apps/extension、apps/api。
- 配置 TypeScript、ESLint、Prettier、Vitest。
- 建立 docs 和基础开发脚本。

验收：

- `pnpm install` 成功。
- `pnpm dev` 可启动本地 Web 和 API。
- `pnpm test` 可运行。

## 里程碑 1：经历库 MVP

目标：

- 实现经历库 CRUD。
- 支持项目、实习、工作、开源、教育经历。
- 支持技能证据、真实性等级和负责程度。

验收：

- 用户能录入完整经历。
- 能按标签和技能搜索经历。
- 数据写入 SQLite。

当前状态：

- 已完成 SQLite repository、API CRUD、Web 端基础录入/编辑/删除/列表。
- 后续可继续补充导入器、批量编辑和更细的技能证据管理。

## 里程碑 2：岗位采集 MVP

目标：

- Chrome 插件读取 Boss 当前页面岗位信息。
- 发送到本地 API。
- Web App 中展示岗位池。

验收：

- 能采集岗位标题、薪资、城市、JD、公司信息。
- 重复岗位能识别。
- 采集失败有错误提示。

当前状态：

- 已完成岗位池 SQLite repository、API CRUD、已保存岗位分析入口。
- Web 端已支持手动保存岗位、查看岗位池、删除岗位和触发匹配分析。
- 插件 popup 已支持在 Boss 页面一键采集当前岗位并保存到本地 API。

## 里程碑 3：AI 岗位分析

目标：

- 实现 JD Parser。
- 实现匹配评分。
- 实现风险识别。
- 展示推荐动作。

验收：

- 每个岗位都有结构化 JD。
- 每个岗位有匹配分和风险点。
- 用户能看到简历强化建议。

当前状态：

- 已完成规则版岗位分析、匹配评分、风险词识别和推荐动作。
- 分析结果已持久化到 `job_analyses`，Web 可展示最新分析。
- 已接入经历库匹配，能输出 `matchedExperienceIds`。
- Web 已将匹配经历展示升级为标题、摘要和技术栈；历史缺失素材会退回显示原始 ID。

## 里程碑 4：个性化投递包

目标：

- 根据岗位生成打招呼语。
- 根据经历库生成简历 Markdown。
- 保存简历版本和改写说明。

验收：

- 每个岗位能生成至少 3 个打招呼语版本。
- 能生成 1 页快投版和 2 页正式版简历。
- 简历能追溯到使用的经历素材。

当前状态：

- 已完成第一版 Markdown 定制简历生成和 `resume_versions` 持久化。
- 已完成第一版打招呼语草稿生成，并与最新简历版本关联为 `application draft`。
- 尚未支持多版本话术、简历在线编辑和导出 PDF/DOCX。

## 里程碑 5：投递管理

目标：

- 投递看板。
- 状态流转。
- 跟进提醒。
- 投递事件时间线。

验收：

- 能记录已打招呼、已投递、已回复、面试、拒绝、Offer。
- 能看到今日待跟进。
- 能查看每个岗位使用的简历和话术。

当前状态：

- 已完成 `applications` 与 `application_events` 表初始化。
- 已支持生成投递草稿，并记录关联简历版本和打招呼语。
- 已支持手动更新投递状态，并记录状态变更事件。
- 尚未支持投递看板、今日跟进提醒和完整事件时间线 UI。

## 里程碑 6：插件辅助填写

目标：

- 插件侧边栏展示当前岗位分析。
- 一键填入打招呼语。
- 用户确认后发送。

验收：

- 插件能从本地 API 获取投递包。
- 能把话术填入页面输入框。
- 发送动作默认由用户手动确认。

## 里程碑 7：效果复盘

目标：

- 统计回复率、面试率。
- 对比简历版本和话术版本效果。
- 给出下一轮策略建议。

验收：

- 能按岗位类型、公司类型、简历版本查看效果。
- AI 能输出策略复盘。
