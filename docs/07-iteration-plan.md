# 迭代计划

## 里程碑 0：项目脚手架

目标：

- 初始化 Git 仓库。
- 创建 pnpm workspace。
- 创建 apps/web、apps/extension、apps/api。
- 配置 TypeScript、ESLint、Prettier、Vitest。
- 建立 docs 和基础开发脚本。

验收：

- `pnpm install` 成功。
- `pnpm dev` 可启动本地 Web 和 API。
- `pnpm test` 可运行。

## 里程碑 1：经历库 MVP

目标：

- 实现经历库 CRUD。
- 支持项目、实习、工作、开源、教育经历。
- 支持技能证据、真实性等级和负责程度。

验收：

- 用户能录入完整经历。
- 能按标签和技能搜索经历。
- 数据写入 SQLite。

当前状态：

- 已完成 SQLite repository、API CRUD、Web 端基础录入/编辑/删除/列表。
- 后续可继续补充导入器、批量编辑和更细的技能证据管理。

## 里程碑 2：岗位采集 MVP

目标：

- Chrome 插件读取 Boss 当前页面岗位信息。
- 发送到本地 API。
- Web App 中展示岗位池。

验收：

- 能采集岗位标题、薪资、城市、JD、公司信息。
- 重复岗位能识别。
- 采集失败有错误提示。

当前状态：

- 已完成岗位池 SQLite repository、API CRUD、已保存岗位分析入口。
- Web 端已支持手动保存岗位、查看岗位池、删除岗位和触发匹配分析。
- 插件 popup 已支持在 Boss 页面一键采集当前岗位并保存到本地 API。

## 里程碑 3：AI 岗位分析

目标：

- 实现 JD Parser。
- 实现匹配评分。
- 实现风险识别。
- 展示推荐动作。

验收：

- 每个岗位都有结构化 JD。
- 每个岗位有匹配分和风险点。
- 用户能看到简历强化建议。

当前状态：

- 已完成规则版岗位分析、匹配评分、风险词识别和推荐动作。
- 分析结果已持久化到 `job_analyses`，Web 可展示最新分析。
- 已接入经历库匹配，能输出 `matchedExperienceIds`。
- Web 已将匹配经历展示升级为标题、摘要和技术栈；历史缺失素材会退回显示原始 ID。

## 里程碑 4：个性化投递包

目标：

- 根据岗位生成打招呼语。
- 根据经历库生成简历 Markdown。
- 保存简历版本和改写说明。

验收：

- 每个岗位能生成至少 3 个打招呼语版本。
- 能生成 1 页快投版和 2 页正式版简历。
- 简历能追溯到使用的经历素材。

当前状态：

- 已完成快投版、正式版和技术版 Markdown 定制简历生成，并写入 `resume_versions` 持久化。
- 已完成礼貌版、证据版和主动版打招呼语生成，并与最新简历版本关联为 `application draft`。
- 已支持从岗位卡片复制包含岗位、分析、匹配经历、简历、打招呼语和状态时间线的完整投递包 Markdown。
- 已支持手动编辑 Markdown 简历并保存为新版本，也支持编辑当前打招呼语草稿。
- 已支持最新和上一版简历、打招呼语的轻量对比与复制；尚未支持完整 diff 和导出 PDF/DOCX。

## 里程碑 5：投递管理

目标：

- 投递看板。
- 状态流转。
- 跟进提醒。
- 投递事件时间线。

验收：

- 能记录已打招呼、已投递、已回复、面试、拒绝、Offer。
- 能看到今日待跟进。
- 能查看每个岗位使用的简历和话术。

当前状态：

- 已完成 `applications` 与 `application_events` 表初始化。
- 已支持生成投递草稿，并记录关联简历版本和打招呼语。
- 已支持手动更新投递状态，并记录状态变更事件。
- 已支持按投递状态聚合数量和筛选岗位的轻量投递看板。
- 已支持在岗位卡片中展示投递状态变更时间线。
- 已支持设置下次跟进时间，并通过“今日待跟进 / 逾期 / 今天 / 未来 3 天”队列筛选待处理岗位。
- 尚未支持自动提醒推送。

## 里程碑 6：插件辅助填写

目标：

- 插件侧边栏展示当前岗位分析。
- 一键填入打招呼语。
- 用户确认后发送。

验收：

- 插件能从本地 API 获取投递包。
- 能把话术填入页面输入框。
- 发送动作默认由用户手动确认。

当前状态：

- 插件 popup 已支持在 Boss 页面提取当前岗位，并匹配本地岗位池中已保存的岗位。
- 插件已支持读取当前岗位最新打招呼语，支持复制话术和尝试填入页面输入框。
- 插件已支持从本地 API 读取当前岗位完整投递包 Markdown，并复制到剪贴板。
- 填入后不会自动发送，仍要求用户检查内容并手动发送。
- 尚未提供完整侧边栏视图，当前先覆盖打招呼语辅助填写和投递包复制闭环。

## 里程碑 7：效果复盘

目标：

- 统计回复率、面试率。
- 对比简历版本和话术版本效果。
- 给出下一轮策略建议。

验收：

- 能按岗位类型、公司类型、简历版本查看效果。
- AI 能输出策略复盘。

当前状态：

- 已完成 Web 岗位池内的第一版投递复盘摘要。
- 已支持查看投递推进率、回复率、面试转化、平均匹配分。
- 已支持提示逾期跟进和推进中但未设置下次跟进的岗位数量。
- 已支持按投递建议、城市和版本迭代数量查看轻量分布。
- 已支持按岗位类型、公司类型、城市、投递建议和具体简历版本做轻量效果归因。
- 已支持基于跟进风险、投递转化、匹配分、简历生成覆盖率和归因信号生成规则版策略建议。
- 已支持按投递状态、投递建议和城市筛选复盘范围。
- 已支持通过本地 API 生成结构化 AI 策略复盘摘要，配置 Provider 后可走真实模型。
- 已预留 OpenAI-compatible AI Provider，默认配置面向 PackyAPI，可通过环境变量启用。
- 策略复盘、岗位分析、简历生成和打招呼语已支持在配置 Provider 后走真实模型；未配置密钥时仍回退规则版。
- Provider 调用失败时，API 会回退到规则版结果并返回用户可见的降级提示。
- Web 投递复盘区已提供 AI Provider 健康检查入口，可确认 PackyAPI 配置、模型调用和网络状态。
- 已支持记录最近 AI 生成请求的运行元数据，并在 Web 投递复盘区展示模型调用、降级和耗时信息。
- Web 投递复盘区已支持按能力、状态和岗位筛选 AI 生成记录，并查看单次运行明细。
