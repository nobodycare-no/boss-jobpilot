# 迭代计划

## V1 阶段结束标识

V1 的阶段名为 **Functional Complete**。只有当里程碑 0-7 的核心验收项全部达到“已完成”并通过一次端到端验收，才算 V1 结束。

2026-06-10 已完成 V1 端到端验收，V1 Functional Complete 通过。后续“继续开发”默认进入 V2 规划或处理真实使用中发现的缺陷，不再把 V1 里程碑作为未完成项反复追加功能。

V1 端到端验收必须覆盖：

- 新手或开发者可启动 API、Web 和插件开发/构建流程。
- 能录入、编辑、删除经历，并按技能、标签或关键词筛选经历。
- 能从 Boss 页面通过插件采集岗位，重复岗位不会产生不可控重复数据。
- 能在 Web 岗位池分析岗位，看到匹配分、风险、匹配经历和简历策略。
- 能生成并手动编辑快投版、正式版、技术版简历，能生成并手动编辑礼貌版、证据版、主动版打招呼语。
- 能复制完整投递包，插件能读取本地话术/投递包并辅助填入，发送仍由用户手动确认。
- 能标记投递状态、设置跟进时间、查看状态时间线和复盘统计。
- `npm run typecheck`、`npm run test`、`npm run lint`、`npm run build`、`codegraph sync .` 全部通过。

## 里程碑 2：岗位采集 MVP

目标：

- Chrome 插件读取 Boss 当前页面岗位信息。
- 发送到本地 API。
- Web App 中展示岗位池。

验收：

- 能采集岗位标题、薪资、城市、JD、公司信息。
- 重复岗位能识别。
- 采集失败有错误提示。

当前状态：

- 已完成岗位池 SQLite repository、API CRUD、已保存岗位分析入口。
- Web 端已支持手动保存岗位、查看岗位池、删除岗位和触发匹配分析。
- 插件 popup 已支持在 Boss 页面一键采集当前岗位并保存到本地 API。
- 插件保存前会按稳定 URL 或“岗位标题 + 公司名”匹配本地岗位池，已存在时只提示“无需重复保存”，不会继续写入重复岗位。
- 插件采集失败时会显示明确错误提示，例如本地 API 未启动、当前页面不是可识别岗位页或 API 保存失败。

## 里程碑 7：效果复盘

目标：

- 统计回复率、面试率。
- 对比简历版本和话术版本效果。
- 给出下一轮策略建议。

验收：

- 能按岗位类型、公司类型、简历版本查看效果。
- AI 能输出策略复盘。

当前状态：

- 已完成 Web 岗位池内的第一版投递复盘摘要。
- 已支持查看投递推进率、回复率、面试转化、平均匹配分。
- 已支持提示逾期跟进和推进中但未设置下次跟进的岗位数量。
- 已支持按投递建议、城市和版本迭代数量查看轻量分布。
- 已支持按岗位类型、公司类型、城市、投递建议和具体简历版本做轻量效果归因。
- 已支持基于跟进风险、投递转化、匹配分、简历生成覆盖率和归因信号生成规则版策略建议。
- 已支持按投递状态、投递建议和城市筛选复盘范围。
- 已支持通过本地 API 生成结构化 AI 策略复盘摘要，配置 Provider 后可走真实模型。
- 已预留 OpenAI-compatible AI Provider，默认配置面向 PackyAPI，可通过环境变量启用。
- 策略复盘、岗位分析、简历生成和打招呼语已支持在配置 Provider 后走真实模型；未配置密钥时仍回退规则版。
- Provider 调用失败时，API 会回退到规则版结果并返回用户可见的降级提示。
- Web 投递复盘区已提供 AI Provider 健康检查入口，可确认 PackyAPI 配置、模型调用和网络状态。
- AI Provider 健康检查失败时会显示当前模型和 base URL，便于区分密钥未配置、模型名无效和网络不可达。
- AI Provider 已兼容 PackyAPI GPT-5.5 的 JSON mode 约束：自动归一化误填 endpoint 的 base URL、补齐用户级 `json` 指令，并默认省略 `temperature`。
- 已支持记录最近 AI 生成请求的运行元数据，并在 Web 投递复盘区展示模型调用、降级和耗时信息。
- Web 投递复盘区已支持按能力、状态和岗位筛选 AI 生成记录，并查看单次运行明细。
