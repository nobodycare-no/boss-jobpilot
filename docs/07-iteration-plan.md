# 迭代计划

## 里程碑 0：项目脚手架

目标：

- 初始化 Git 仓库。
- 创建 pnpm workspace。
- 创建 apps/web、apps/extension、apps/api。
- 配置 TypeScript、ESLint、Prettier、Vitest。
- 建立 docs 和基础开发脚本。

验收：

- `pnpm install` 成功。
- `pnpm dev` 可启动本地 Web 和 API。
- `pnpm test` 可运行。

## 里程碑 1：经历库 MVP

目标：

- 实现经历库 CRUD。
- 支持项目、实习、工作、开源、教育经历。
- 支持技能证据、真实性等级和负责程度。

验收：

- 用户能录入完整经历。
- 能按标签和技能搜索经历。
- 数据写入 SQLite。

## 里程碑 2：岗位采集 MVP

目标：

- Chrome 插件读取 Boss 当前页面岗位信息。
- 发送到本地 API。
- Web App 中展示岗位池。

验收：

- 能采集岗位标题、薪资、城市、JD、公司信息。
- 重复岗位能识别。
- 采集失败有错误提示。

## 里程碑 3：AI 岗位分析

目标：

- 实现 JD Parser。
- 实现匹配评分。
- 实现风险识别。
- 展示推荐动作。

验收：

- 每个岗位都有结构化 JD。
- 每个岗位有匹配分和风险点。
- 用户能看到简历强化建议。

## 里程碑 4：个性化投递包

目标：

- 根据岗位生成打招呼语。
- 根据经历库生成简历 Markdown。
- 保存简历版本和改写说明。

验收：

- 每个岗位能生成至少 3 个打招呼语版本。
- 能生成 1 页快投版和 2 页正式版简历。
- 简历能追溯到使用的经历素材。

## 里程碑 5：投递管理

目标：

- 投递看板。
- 状态流转。
- 跟进提醒。
- 投递事件时间线。

验收：

- 能记录已打招呼、已投递、已回复、面试、拒绝、Offer。
- 能看到今日待跟进。
- 能查看每个岗位使用的简历和话术。

## 里程碑 6：插件辅助填写

目标：

- 插件侧边栏展示当前岗位分析。
- 一键填入打招呼语。
- 用户确认后发送。

验收：

- 插件能从本地 API 获取投递包。
- 能把话术填入页面输入框。
- 发送动作默认由用户手动确认。

## 里程碑 7：效果复盘

目标：

- 统计回复率、面试率。
- 对比简历版本和话术版本效果。
- 给出下一轮策略建议。

验收：

- 能按岗位类型、公司类型、简历版本查看效果。
- AI 能输出策略复盘。
